YAXArrays.jl

Yet another xarray-like Julia package

A package for operating on out-of-core labeled arrays, based on stores like NetCDF, Zarr or GDAL.

Get Started
View on Github
API reference

Flexible I/O capabilities

Open and operate on NetCDF and Zarr datasets directly. Or bring in data from other sources with ArchGDAL.jl, GRIBDatasets.jl, GeoJSON.jl, HDF5.jl, Shapefile.jl, GeoParquet.jl, etc.

Interoperability

Well integrated with Julia's ecosystem, i.e., distributed operations are native. And plotting with Makie.jl is well supported.

Named dimensions and GroupBy(in memory)

Apply operations over named dimensions, select values by labels and integers as well as efficient split-apply-combine operations with groupby via DimensionalData.jl.

Efficiency

Efficient mapslices(x) and mapCube operations on huge multiple arrays, optimized for high-latency data access (object storage, compressed datasets).

How to Install YAXArrays.jl? ​

Since YAXArrays.jl is registered in the Julia General registry, you can simply run the following command in the Julia REPL:

julia
julia> using Pkg
julia> Pkg.add("YAXArrays.jl")
# or
julia> ] # ']' should be pressed
pkg> add YAXArrays

If you want to use the latest unreleased version, you can run the following command:

julia
pkg> add YAXArrays#master

Want interoperability? ​

Install the following package(s) for:

using Pkg
